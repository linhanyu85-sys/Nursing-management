# AI 护理后台管理系统

这个目录现在是项目正式的后台管理端，不再只是预留骨架。

## 目标

后台管理系统直接接入主项目 `api-gateway`，用于：

- 管理软件账号，并同步登录服务与协同服务
- 编辑病例、床位、风险标签、待办与观察值
- 查看病区可视化分析看板
- 监控设备会话、网关状态与实时床位流

## 数据互联方式

- 病例与病区数据：写入主 PostgreSQL 业务表
- 软件账号：同步写入 `auth-service` 与 `collaboration-service`
- 软件端：继续通过原有后端接口读取，因此后台改动后软件端会看到相同数据

## 运行方式

### 1. 作为主系统管理端

启动后端后，直接访问：

- `http://127.0.0.1:18000/admin/`

这是推荐方式。页面与 API 共用同一个网关入口。

### 2. 本地静态预览

如果只想预览页面结构，可以运行：

```powershell
cd "D:\Desktop\ai agent 护理精细化部署\apps\admin-web"
.\start_web.ps1 -Port 8099
```

然后在页面右上角“连接配置”里把 API 地址改成你的网关地址。

## 文件说明

- `index.html`：后台主页面
- `assets/styles.css`：视觉样式
- `assets/app.js`：后台逻辑、数据加载、编辑与保存
- `start_web.ps1`：本地静态预览脚本
